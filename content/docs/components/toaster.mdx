## Introduction

The Toaster component provides a powerful, accessible toast notification system built on top of Sonner. It seamlessly integrates with your theme system and design tokens, offering a consistent way to display temporary messages, alerts, and feedback to users.

### Key Features

- **Built on Sonner**: Industry-standard toast library with excellent performance
- **Theme Integration**: Automatically adapts to light/dark mode
- **Multiple Types**: Support for default, success, error, warning, and info toasts
- **Rich Content**: Descriptions, actions, and custom JSX content
- **Promise Support**: Built-in loading states for async operations
- **Positioning**: Six position options (top/bottom × left/center/right)
- **Accessible**: Full keyboard navigation and screen reader support
- **Mobile-Friendly**: Swipe to dismiss gestures
- **Auto-dismiss**: Configurable duration with hover-to-pause
- **Stack Management**: Smart stacking of multiple toasts

## Usage

First, add the `Toaster` component to your app layout:

```tsx
import { Toaster } from "@mostaphaqammari/soar-ds";

export default function RootLayout({ children }) {
  return (
    <html>
      <body>
        {children}
        <Toaster />
      </body>
    </html>
  );
}
```

Then use the `toast` function anywhere in your app:

```tsx
import { toast } from "@mostaphaqammari/soar-ds";

function MyComponent() {
  return <button onClick={() => toast("Event has been created")}>Show Toast</button>;
}
```

## Accessibility

The Toaster component is built with accessibility in mind:

- **Screen Readers**: Toasts are announced using `role="status"` for informational messages and `role="alert"` for errors
- **Keyboard Navigation**: Press `Escape` to dismiss toasts
- **Focus Management**: Focus is properly managed when actions are present
- **ARIA Labels**: Proper labeling for all interactive elements
- **Reduced Motion**: Respects `prefers-reduced-motion` setting
- **Color Contrast**: Meets WCAG AA standards

### Best Practices

1. **Use Appropriate Types**: Match toast type to message severity
2. **Keep Messages Brief**: Short, actionable messages work best
3. **Avoid Overuse**: Don't overwhelm users with too many toasts
4. **Provide Actions**: Include undo/retry actions when appropriate
5. **Set Reasonable Durations**: 4-6 seconds for reading, longer for complex messages
6. **Don't Rely on Color Alone**: Use icons and text to convey meaning

## Usage Guidelines

### When to Use

- ✅ Confirming user actions (save, delete, update)
- ✅ Displaying non-critical errors
- ✅ Showing progress of async operations
- ✅ Providing feedback without disrupting workflow
- ✅ Temporary status updates

### When Not to Use

- ❌ Critical errors requiring immediate attention (use Dialog)
- ❌ Complex information requiring user input (use Dialog)
- ❌ Permanent status indicators (use Badge)
- ❌ Navigation feedback (use browser URL)
- ❌ Form validation errors (use inline validation)

## Common Patterns

### Form Submission Feedback

```tsx
import { toast } from "@mostaphaqammari/soar-ds";

async function handleSubmit(data: FormData) {
  const promise = submitForm(data);

  toast.promise(promise, {
    loading: "Saving changes...",
    success: "Changes saved successfully",
    error: "Failed to save changes",
  });
}
```

### Undo Action

```tsx
import { toast } from "@mostaphaqammari/soar-ds";

function handleDelete(id: string) {
  const backup = data.find((item) => item.id === id);
  deleteItem(id);

  toast("Item deleted", {
    action: {
      label: "Undo",
      onClick: () => {
        restoreItem(backup);
        toast.success("Item restored");
      },
    },
  });
}
```

### Persistent Loading with Update

```tsx
import { toast } from "@mostaphaqammari/soar-ds";

async function uploadFile(file: File) {
  const toastId = toast.loading("Uploading file...");

  try {
    await upload(file);
    toast.success("File uploaded successfully", { id: toastId });
  } catch (error) {
    toast.error("Failed to upload file", { id: toastId });
  }
}
```

### Multiple Toasts with Priority

```tsx
import { toast } from "@mostaphaqammari/soar-ds";

// Important toast that doesn't auto-dismiss
toast.error("Critical error occurred", {
  important: true,
  duration: Infinity,
});

// Regular toast
toast.info("New message received");
```

## Toaster vs Alert vs Alert Dialog

Understanding when to use each notification component:

| Feature          | Toaster                  | Alert                       | Alert Dialog                  |
| ---------------- | ------------------------ | --------------------------- | ----------------------------- |
| **Persistence**  | Temporary (auto-dismiss) | Permanent (until dismissed) | Temporary (until user action) |
| **Interruption** | Non-blocking             | Non-blocking                | Blocking                      |
| **Position**     | Floating (edges/corners) | Inline (flow)               | Center overlay                |
| **User Action**  | Optional                 | Optional                    | Required                      |
| **Use Case**     | Feedback, status updates | Warnings, information       | Critical confirmations        |
| **Dismissal**    | Auto or manual           | Manual                      | Manual (action required)      |

**Choose Toaster** when you need to provide temporary feedback without interrupting the user's workflow.

**Choose Alert** when you need to display persistent, non-blocking information within the page flow.

**Choose Alert Dialog** when you need to interrupt the user and require their explicit action before proceeding.

## Advanced Usage

### Global Configuration

Configure default options for all toasts:

```tsx
<Toaster position="top-center" expand visibleToasts={5} duration={5000} closeButton />
```

### Custom Icons

Override default icons per type:

```tsx
import { CheckCircle } from "lucide-react";

toast.success("Operation completed", {
  icon: <CheckCircle />,
});
```

### Queue Management

Control how toasts are queued:

```tsx
// Dismiss all toasts
toast.dismiss();

// Dismiss specific toast
const id = toast("Hello");
toast.dismiss(id);

// Update existing toast
toast.success("Updated!", { id });
```

### Get Toast History

Access toast history and active toasts:

```tsx
import { toast } from "@mostaphaqammari/soar-ds";

// Get all toasts (history)
const history = toast.getHistory();

// Get active toasts
const active = toast.getToasts();
```

## Performance Tips

1. **Batch Operations**: Don't show a toast for each item in a bulk operation; show one summary toast
2. **Debounce Rapid Events**: Prevent toast spam from rapid user actions
3. **Use Promise Toasts**: Let the toast system manage loading states automatically
4. **Limit Visible Toasts**: Keep `visibleToasts` low (3-5) for better performance
5. **Cleanup on Unmount**: Dismiss toasts when navigating away from critical flows

```tsx
// Good: Summary toast
toast.success(`${count} items deleted`);

// Bad: Toast per item
items.forEach((item) => toast.success(`${item.name} deleted`));
```

## Examples in Context

### E-commerce Checkout

```tsx
import { toast } from "@mostaphaqammari/soar-ds";

function CheckoutButton() {
  async function handleCheckout() {
    const promise = processPayment();

    toast.promise(promise, {
      loading: "Processing payment...",
      success: "Order placed successfully!",
      error: "Payment failed. Please try again.",
    });
  }

  return <button onClick={handleCheckout}>Complete Order</button>;
}
```

### File Upload

```tsx
import { toast } from "@mostaphaqammari/soar-ds";

function FileUploader() {
  async function uploadFile(file: File) {
    const toastId = toast.loading(`Uploading ${file.name}...`);

    try {
      await uploadToServer(file);
      toast.success(`${file.name} uploaded successfully`, { id: toastId });
    } catch (error) {
      toast.error(`Failed to upload ${file.name}`, { id: toastId });
    }
  }

  return <input type="file" onChange={(e) => uploadFile(e.target.files[0])} />;
}
```

### Social Features

```tsx
import { toast } from "@mostaphaqammari/soar-ds";

function LikeButton({ postId }: { postId: string }) {
  function handleLike() {
    likePost(postId);

    toast("Post liked", {
      icon: "❤️",
      action: {
        label: "Undo",
        onClick: () => {
          unlikePost(postId);
          toast("Like removed");
        },
      },
    });
  }

  return <button onClick={handleLike}>Like</button>;
}
```
